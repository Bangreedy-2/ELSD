Compiler Pipeline
How a .gg source file is transformed into G-code, step by step.
What this diagram shows

The full compilation path from a human-readable .gg file to machine-ready G-code across 5 distinct stages.

  • Lexer — tokenises source text into a flat token stream. Throws SyntaxError on unknown characters. (GGCodeLexer.py)
  • Parser — validates syntax and builds a concrete parse tree from the token stream. (GGCodeParser.py)
  • AST Builder — converts the parse tree into typed Python objects: MoveStatement, RepeatStatement, etc. (antlr_converter.py)
  • Semantic Analyzer — resolves variable scopes, checks types, throws SemanticError on violations. (semantic_analyzer.py)
  • Code Generator — traverses the validated AST and emits G-code. Tracks Z-axis moves to insert layer annotation comments automatically. (gcode_generator.py)
  • Output — plain-text G-code string: G0, G1, G2/G3, M104, G28, G4 …
Diagram 2 of 3
System Architecture
Three-tier layout: browser client, HTTP API server, and the compiler core.
What this diagram shows

The project is split into three independent layers. Frontend and backend communicate over HTTP; the compiler core is a pure Python library with no network code of its own.

  • Frontend (React 18 + TypeScript, Vite, port 5173) — code editor, sample file picker, 3D WebGL viewer (Three.js / OrbitControls), G-code output panel.
  • Backend (FastAPI + uvicorn, port 8000) — POST /compile: accepts {code, geometry_mode}, returns {gcode}. POST /upload: accepts a multipart file, returns its text content.
  • Compiler Core (src/) — run_pipeline() chains all 5 stages. The geometry_mode parameter switches between polyline approximation (segmented) and true G2/G3 arcs (exact).

GG Language Reference
All built-in commands grouped by category, with a side-by-side compilation example.
What this diagram shows

The top half organises every built-in command into four categories. The bottom half shows a .gg code snippet next to the G-code it produces after compilation.

  • Motion — Move to X Y, Move to Z, Move to X Y Z (triplet), Move to Origin, Home. Emit G1 / G28. Z moves trigger automatic layer annotation comments.
  • Geometry — Add Square, Add Rectangle, Add Circle, Add Line. Circles: 32-point polylines in segmented mode, or a single G2/G3 arc in exact mode.
  • Machine Control — Temperature → M104, Wait for N seconds/minutes → G4, Set speed to N (feed rate), Set layer_height to N (tracked in gcode_generator.py).
  • Control Flow & Variables — If/else, Repeat N times, While, Define / Set. Arithmetic: + - * /. Comparisons: == != > >= < <=. Variables are block-scoped.
